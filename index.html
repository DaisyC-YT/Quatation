03WORKFLOW // 專案溝通與執行流程
01
PHASE_01
⏱ 1–2 DAYS
需求訪談與規劃
收集目標市場、風格參考、功能需求（線上諮詢表、多語系、案例篩選），以及內容現況（文案與照片是否自備）。確認技術選型（WordPress 或 Astro）與是否需要自製設計稿。
02
PHASE_02
⏱ 3–5 DAYS
設計方向確認
以 Relume 或 Framer 產出 2–3 個視覺方向，涵蓋首頁、案例列表、案例內頁、關於、聯絡表單。客戶確認後才進入開發，避免後期大規模修改。
03
PHASE_03
⏱ 10–20 DAYS
網站開發
建立頁面結構、整合設計稿、建立案例後台（WordPress CPT 或 Astro Content Collections）、表單整合（Formspree / EmailJS，避免安裝重量外掛）。
04
PHASE_04
⏱ 2–3 DAYS
SEO 基礎設定
最常被接案者忽略卻最值錢的環節，完整執行以下清單：
  • 安裝 Rank Math（WordPress）或手動設定 <meta> / Open Graph（Astro）
  • 設定 XML Sitemap 並提交 Google Search Console
  • 建立 robots.txt
  • 加入 JSON-LD 結構化資料（LocalBusiness、Service、FAQPage）
  • 設定 Google Analytics 4 + 事件追蹤（表單提交、電話點擊）
  • 圖片全部轉 WebP、設定 lazy loading 與 alt 文字
  • 設定 canonical URL，解決重複頁面問題
  • 啟用 Cloudflare 免費方案（CDN + 基礎 DDoS 防護）
05
PHASE_05
⏱ 依客戶速度
內容上架
協助客戶上稿或進行教學。
06
PHASE_06
⏱ 2–3 DAYS
測試與優化
PageSpeed Insights 驗證 Core Web Vitals 達標（LCP < 2.5s），進行跨瀏覽器測試、行動裝置測試、表單送出測試，確保上線品質穩定。
07
PHASE_07
⏱ 交付日
交付與維護合約
完成操作教學、帳號移交、說明文件交付。視套案等級，評估是否簽訂後續維護合約（月費制）。
